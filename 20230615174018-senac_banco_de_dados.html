Bancos de dados: introdução a banco de dados relacional, funcionalidade, Sistema Gerenciador de Banco de Dados (SGBD), implantação do banco de dados

Bancos de dados são caracterizados pela eficiência, escalabilidade, confiabilidade, abstração dedados, persistência e muitas outras qualidades técnicas que os tornam um recurso essencial em praticamente qualquer projeto de software que trabalhe com dados.

Introdução a banco de dados relacional

  • Um banco de dados é um local onde são armazenados dados necessários para a manutenção e persistencia de atividades. Funciona como um repositório de fonte de dados para aplicações atuais e para outras que vierem a existir
  • É possivel armazenar dados em um computador de duas maneiras
    1. Utilizando arquivos de dados permanentes
    2. Utilizando um banco de dados

Banco de dados vs Arquivos de dados permanentes

  • Na abordagem por arquivos de dados permanentes é como se o programa criasse ou editasse um ou mais arquivos de texto, onde aloca informações
    • Isso pode ser o suficiente para alguns tipos de software
      • Configurações básicas de programas (.ini, .config, filesrc, etc)
    • A abordagem de bancos de dados é, na maioria das vezes, mais adequada
  • A abordagem por bancos de dados possui as seguintes vantagens
    • Controle centralizado de dados:
      • Aloca todos os dados em um unico local, proporcionando maior controle sobre eles. Fonte unica de verdade.
      • Dados armazenados em arquivos estão disperços
    • Controle de redundancia, redução do espaço de armazenamento e compartilhamento de dados
      • Existe no modo por arquivos um desperdicio de espaço de armazenamento, visto que a mesma informação pode aparecer em dois lugares diferentes, já no banco de dados uma mesma informação pode ser compartilhada por diversas aplicações
    • Eliminação de incônsistencias e garantia de integridade
      • O banco de dados ajuda a eliminar inconscistencias entre dois arquivos que possuem a mesma informação porém com valores diferentes.
    • Estabelecimento de padrões e facilidade de acesso aos dados
      • Como o banco centraliza todos os dados ele é mais acertivo em definir padrões, numenclaturas e documentações. Sendo assim é possivel padronizar a forma de consulta e alteração de dados, enquanto em um sistema baseado em arquivos essa forma pode variar de forma mais imprevisivel
    • Independencia de dados
      • No sistema de arquivos a definição da estrutura dos dados e método de acesso são coodependentes, tornando impossivel alterar a forma de consulta sem alterar a estrutura do arquivo. Um banco de dados, por sua vez de independencia de dados, isso é, os dados são guardados de maneira independente da aplicação que os consulta, tornando extremamente flexivel a forma de consulta.

Funcionalidade

  • Entendendo o funcionamento de um banco de dados de uma perspectiva mais intuitiva:

    • O Banco de dados é como um quarto que contém varias caixas para armazenar diversos objetos. Sendo o banco de dados o quarto e cada caixa uma tabela. Cada caixa possue um nome identificador, que a torna unica. A partir dessa estrutura de nomes unicos é possivel armazenar dados diferentes em tabelas diferentes.

  • Bancos de dados relacionais baseam-se em tabelas constituidas por colunas, cada uma contendo um tipo especifico de informação.

    • Exemplo: Tabela - Pessoas

    | id | nome | idade | pais |

    1 Henrique Marques 28 Brasil
    2 Pedro Nunes 35 Portugal

    • Dessa forma cada tebela tem um assunto de interesse (separation of concerns) e denotarão alguma informação especifica que deve ser armazenada.

      Por exemplo, para um banco de dados de uma loja, você poderia ter uma tabela para "cliente", que guardaria os diferentes dados de um cliente (nome, endereço, idade, entre outros), ou uma tabela para "venda", com informações como a data da venda, valor total etc.

Relações

  • Normalmente tabelas n ficam sozinhas, elas se ligam umas as outras através de relações.
  • Exemplo de relação entre tabelas

    • Tabela A) Clientes:

      id nome endereço idade
      1 João Rua A, 10 35
      2 Maria Rua B, 50 70
      3 Joaquim Rua C, 5 20

    • Tabela B) Vendas:

      id datavenda valortotal idcliente
      1 10/10/21 1000 2
      2 11/10/21 150 1
      3 11/10/21 79 2
      • Perceba que a tabela B se relaciona com a tabela A através do campo idcliente (foreing key), sendo assim sabemos que as compras de id 1 e 3 foram feita pelo cliente de id 2 (Maria) e a compra de id 2 foi feita pelo cliente de id 1 (João)
  • No exemplo acima percebemos que não é necessário repetir todas as informações do cliente para cada venda, já que podemos inferir o cliente atráves do campo idcliente cadastrado na tabela B. Dessa forma criamos uma relação entre tabelas
  • Essas tebalas não vem prontas e precisa ser modeladas especificamente para cada modalidade de negocio. Após construção do banco entre suas funcionalidades estarão as de:
    • Incluir novas informações de forma centralziada
    • Atualizar ou remover informações especificas
    • Repucperar informações de uma ou várias tabelas
    • Aplicar regras de segurança para acesso às informações
  • Todas essas tarefas serão realizadas com agilidade pelo sistema de banco de dados, permitindo a manipulação de uma grande quantidade de informações.

Modelo relacional x Não relacional

  • Bancos de dados relacionais não são os unicos modelos de banco de dados, também há sua variação não relacional, sendo a mais comum o NoSQL.
  • É necessário avaliar as especificações de uma aplicação para decidir entre um banco relacional ou não relacional

Como funciona um banco relacional:

  • Banco de dados relacional fornecem uma maneira padrão de representar e consultar dados que pode ser utilizada por qualquer aplicativo. O modelo relacional brilha no uso das tabelas que são modeladas e estruturadas. No modelo relacional usa como forma de consulta o SQL (strutured query language). O SQL padroniza a forma de consulta de dados e resolve o problema de estruturas de dados arbitrárias. Em cada tabela de um banco relacional há uma Chave Primária

  1. Chave primária (primary key):

    • A chave primária é um campo de identificador único de uma determinada inserção em uma tabela, que funciona como fonte unica de verade para uma determinada entrada em uma tabela. É atraves dela que podemos fazer relações, utilizando a chave primária de uma tabela como chave estrangeira (foreign key) em uma outra tabela.

  2. Chave estrangeira (foreign key)

    • A foreign key é uma referencia a uma outra entrada em outra tabela do banco de dados. Nos exemplos acima o campo idcliente é uma foreign key que referencia uma primary key de outra tabela, no caso a tabela cliente, e portanto referencia um cliente especifico. Enquanto uma primary key precisa ser unica dentro de uma tabela uma foreign key pode aparecer diversas vezes em uma tabela, como no caso da Tabela B onde o campo idcliente é preenchido com o id "2" em duas ocasiões, indicando que duas compras foram feitas pelo mesmo cliente.

Como funciona o modelo Não Relacional

  • Em um banco não relacional as tabelas não são utilizadas. Nesse caso o NoSQL é a abordagem mais conhecida. Apesar de se interpretar que nesse caso não há relações entre informações a verdade é que esses bancos podem sim fazer relações, porem de uma maneira diferente.
  • Os bancos NoSQL surgiram para facilitar e diminuir complexidade de modelos de gerenciamento de dados. Geralmente são uteis em situações de volumes muito grandes de dados com enfase na disponibilização de dados recentes.

Existem diversos tipos de banco de dados NoSQL e a principal diferença entre cada um deles é a sua maneira de armazenar os dados. Entre os mais utilizados está o chamado "banco de documentos", que armazena seus dados em documentos semelhantes ao JSON (JavaScript Object Notation).

  • Veja o exemplo de documento JSON a seguir:

    {
   "cep": "01001-000",
   "logradouro": "Praça da Sé",
   "complemento": "lado ímpar",
   "bairro": "Sé",
   "localidade": "São Paulo",
   "uf": "SP",
   "ibge": "3550308",
   "gia": "1004"
}

SGBD e implementação de banco de dados

Sistema gerenciador de Banco de Dados (SGBD) e implementação de banco de dados

  • Um banco de dados pode ser usado em vários contextos. Para isso é interessante ter um SGBD:

    Um SGBD é um software genérico para manipular banco de dados. Ele permite a definição, a construção e o manejo de um banco de dados para diversas aplicações. Durante o curso, você trabalhará com o MySQL, utilizando o MySQL Server e o MySQL Workbench nas edições Community.
