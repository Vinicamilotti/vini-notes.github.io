Manipulação de dados

  • A manipulação de dados é o processo de adicionar, consultar, deletar ou alterar dados.
  • Em bancos de dados relacionais o a manipulção de dados é feita através de SQL.
    • O grupo de comandos utilizados para manipulação de dados recebe o nome de DML – Data manipulation language)

Comandos básicos

  • SELECT para coleta de dados
  • INSERT para inserção de dados
  • UPDATE para alteração de dados já existentes
  • DELETE para deleção de dados

Disclamer

  • O conteudo da aula focará nesses comandos no entanto em alguns momentos será necessário consultar a tabela, para fins demonstrativos.

  • Para consultar tabelas o comando utilizado é o SELECT, que será visto mais a fundo nos textos Consulta de dados e Consulta com múltiplas tabelas.

  • Para os fins dessas anotações utilizar o comando:

    SELECT * FROM nome_da_tabela;

    DROP DATABASE IF EXISTS senacEstudos;

Comandos INSERT, UPDATE, e DELETE

INSERT - Inclusão de dados

  • A instrução INSERT INTO é utilizada para inserir dados em uma tabela e permite fazer isso utilizando diversos parametros variaveis

  • A sintaxe básica do comando INSERT é

    INSERT INTO nome_da_tabela VALUES (valores);

  • Destrinchando:

    1. INSERT: inicia o comando INSERT
    2. INTO: indica que uma tabela será selecionada para inclusão de dados
    3. nomedatabela: tabela selecionada
    4. VALUES: indica quais valores devem ser inseridos, os valores a serem inseridos ficam entre parentenses — (valores)

  1. Tipos de dados a serem inseridos

    • Dados numericos: Devem ser inseridos diretamente
    • Strings: Devem ser inseridos entre '' ou ""
    • Null: deve ser inserido com o token null sem aspas

  2. Inserindo dados

    • Apesar de não ser necessário, é boa prática nomear as colunas a serem preenchidas. Isso garante duas coisas:

      1. Legibilidade do script

      2. Que a ordem em que as colunas estão gravadas no banco não importa, desde que elas sejam preenchidas no Script

        INSERT INTO nome_tabela(coluna1, coluna2, coluna3) VALUES (valor1, valor2, valor3);

      Ou

      INSERT INTO nome_tabela(conluna3, coluna1, coluna2) VALUES (valor3, valor1, valor2);

    • De todo módo é importante se atentar aos nomes dos campos e quantidades de valores indicados na tabela, também é importante saber de quais tipos são os campos listados e inserir valores de tipo que correspondem ao campo

    INSERT INTO nome_tabela(colunaInteiro, colunaSTRING, colunaDATE) VALUES (1, 'Isso é uma string', '2023-06-28');

  3. Formas do insert - Utilizando o banco de dados criado

    1. Insert sem colunas especificadas

      INSERT INTO curso VALUES (
  1,
  'Técnico em Desenvolvimento de Sistemas',
  'Prepara o aluno para exercer funções técnicas, como: administração do sistema de banco de dados, desenvolver software, programar rotinas de sistema utilizando linguagens e técnicas de programação, programação de sistemas para desktop e para web, entre outras atividades.'
  );
      • Dessa forma é necessário saber a ordem das colunas registrada na tabela. Nesse caso, sabemos que a ordem é:
        1. id
        2. nome
        3. descricao
      • Perceba que dessa forma, além de ser necessário seguir a ordem da tabela, precisamos especificar o campo id, apesar de ser um AUTOINCREMENT, pois no caso em que não especificamos quais campos queremos preencher TODOS os campos devem ser especificados, de outra forma o banco não saberia onde alocar cada valor

    2. Insert com colunas especificadas

      • Forma mais comum e eficiente de fazer o INSERT, especificando os campos entre parenteses
      INSERT INTO curso(nome) VALUES ('Técnico em Informática para Internet');
      • Repare que, no nosso banco o valor do campo "descricao" pode ser null
      • Repare também que no campo acima, como especificamos o campo que queremos preencher outros campos da tabela serão preenchidos automaticamente. No caso o id sera auto incrementado e a descrição será NULL. Repare na consulta a seguir os valores:
      SELECT * FROM curso
      id nome descricao
      1 Técnico em Desenvolvimento de Sistemas Prepara o aluno para exercer funções técnicas, como: administração do sistema de banco de dados, desenvolver software, programar rotinas de sistema utilizando linguagens e técnicas de programação, programação de sistemas para desktop e para web, entre outras atividades.
      2 Técnico em Informática para Internet NULL
      • IMPORTATE: A coluna só sera preenchida automáticamente se o campo não for especificado como NOT NULL, se possuir algum DEFAULT ou se for AUTOINCREMENT — Ver Senac: Definição de dados. Se houver outros campos obrigatórios na tabela, mas que não foram especificados no script de inserção o SQL retorná um erro.

  5. Valores de data e hora

    • Para inserir data e hora basta utilizar o formato 'YYYY-MM-DD' para datas (DATE), 'HH:MM:SS'(TIME) para horas e 'YYYY-MM-DD HH:MM:SS' para DATETIME
      • É importante frizar que o tanto data, quanto hora, quanto datahora devem ser mencionados entre '' ou seja, uma string
    INSERT INTO aluno (nome, sobrenome, data_nascimento) VALUES ('Alice', 'Gonçalves', '1998-10-18');
    • Verificando:
    SELECT * FROM aluno
    id nome sobrenome datanascimento cursoid
    1 Alice Gonçalves 1998-10-18 NULL
    • Outro formato de DATE
      • O formato DATE também pode ser escrito como YY-MM-DD
    INSERT INTO aluno (nome, sobrenome, data_nascimento) VALUES ('Eduardo', 'Machado', '98-07-16');
    • No entanto o valor de YY sera convertido para YYYY
    SELECT * FROM aluno
    id nome sobrenome datanascimento cursoid
    1 Alice Gonçalves 1998-10-18 NULL
    2 Eduardo Machado 1998-07-16 NULL
    • Da mesma forma o mês ou dia pode ser escrito com apenas um digito, mas será convertido para 2 automáticamente
    INSERT INTO aluno (nome, sobrenome, data_nascimento) VALUES ('Vitória', 'Prestes', '1997-2-15');
    • Verificando:
    SELECT * FROM aluno
    id nome sobrenome datanascimento cursoid

  6. Inserção de registros com chaves estrangeiras

    Nos exemplos anteriores, foram cadastrados alguns alunos na tabela, mas não foram inseridos valores nos campos "cursoid". Isso porque o campo "cursoid" é um campo que referencia uma chave primária (primary key) de outra tabela. Ou seja, trata-se de um campo com chave estrangeira (foreign key).

    • Tabelas com chave estrangeiras são chamadas de "tabelas filhas", enquanto a tabela com a chave primária referenciada é a "tabela pai".

    • No caso de estudo a tabela aluno é filha da tabela curso

    • Se executarmos o comando

      SELECT id, nome FROM curso;
      id nome
      1 Técnico em Desenvolvimento de Sistemas
      2 Técnico em Informática para Internet
      3 Técnico em Administração
      4 Técnico em Contabilidade
      5 Técnico em Design de Interiores

      • Peceba que cada registro tem um identificador único em "id"

      • Sendo assim podemos adicionar um desses "id"s como valor do campo "cursoid" na tabela Aluno:

        INSERT INTO aluno (nome, curso_id) VALUES ('Alex', 1);

      • Verificando:

        SELECT * FROM aluno

    • A entrada 4 possui como valor de "cursoid" o numero 1, indicando portanto que esse campo referencia a entrada de "id" 1 na tabela curso (Técnico em Desenvolvimento de Sistemas)

    • Perceba no entanto que se adicionarmos um valor não-existente em curso como valor do campo "cursoid" um erro será gerado

    INSERT INTO aluno (nome, curso_id) VALUES ('Ana', 6);

    • O resultado desse comando é:

      ERROR 1452 (23000) at line 2: Cannot add or update a child row: a foreign key constraint fails (`senacEstudo`.`aluno`, CONSTRAINT `aluno_ibfk_1` FOREIGN KEY (`curso_id`) REFERENCES `curso` (`id`))

Atualização de dados — UPDATE

  • O comando UPDATE é responsável por atualizar dados já armazenados em uma tabela de banco de dados.

    Esse comando tem uma das funções mais importantes da linguagem SQL: manter os dados atualizados. Porém, por manipular diretamente dados já existentes no banco de dados, o uso incorreto desse comando pode resultar em perdas de dados significativos. Por isso, é preciso muito cuidado na execução desse comando.

  • Comando update básico

    UPDATE nome_da_tabela SET coluna1 = novo_valor1, coluna2 = novo_valor2 WHERE condição

  • Destrinchando:

    • UPDATE: indica o update e seleciona a tabela a ser modificiada (nomedatabela)
    • SET: indica quais colunas e valores devem ser editados (coluna1=novovalor1, coluna2=novovalor2)
    • WHERE denota em quais condições a alteração deve ser feita, condições podem ser concatenadas com os operadores AND ou OR

  • Exemplificando:

    • Exemplo 1:

      UPDATE aluno SET curso_id = 2 WHERE id = 1

    • Verificando:

      SELECT * FROM aluno
    id nome sobrenome datanascimento cursoid
    1 Alice Gonçalves 1998-10-18 2
    2 Eduardo Machado 1998-07-16 NULL
    3 Vitória Prestes 1997-02-15 NULL
    4 Alex NULL NULL 1

    • Exemplo 2:

      UPDATE aluno SET curso_id = 3 WHERE id = 2 OR nome = "Vitória"

    • Verificando:

      SELECT * FROM aluno
    id nome sobrenome datanascimento cursoid
    1 Alice Gonçalves 1998-10-18 2
    2 Eduardo Machado 1998-07-16 3
    3 Vitória Prestes 1997-02-15 3
    4 Alex NULL NULL 1

    • Exemplo 3:

      UPDATE aluno SET sobrenome = "Telles", data_nascimento = '1992-12-15' WHERE sobrenome IS NULL AND data_nascimento IS NULL

    • Verificando

      SELECT * FROM aluno

  1. Update em uma unica coluna

    • Segue a forma

      UPDATE nome_tabela SET coluna1 = valor1 WHERE condição

  2. Update em várias colunas

    • Mesma formula porém com pares coluna-valor separados por virgula

      UPDATE nome_tabela SET coluna1 = valor1, coluna2 = valor2 WHERE condição

  3. Update em TODOS os registros

    • A clausula WHERE no comando update é opcional, sendo assim é possivel não especificar condição alguma para o comando, fazendo com que ele execute as alterações em todos os registros da tabela
    UPDATE curso SET descricao = 'Em breve...'

    • Verificando:

      SELECT * FROM curso
      id nome descricao
      1 Técnico em Desenvolvimento de Sistemas Em breve…
      2 Técnico em Informática para Internet Em breve…
      3 Técnico em Administração Em breve…
      4 Técnico em Contabilidade Em breve…
      5 Técnico em Design de Interiores Em breve…
    • IMPORTANTE: O Mysql Workbench, por padrão, vem com uma opção de segurança habilitada que não permite a alteração ou deleção de diversas linhas ou tabelas ao mesmo tempo. Para desabilitar essa configuração siga Edit > Preferences > SQL Editor e desmarque a caixa Safe Updates e depois Reconecte ao banco. Exclusão de dados — DELETE

  • O comando DELETE exclui uma linha de uma tabela, eliminando um registro no banco de dados.

  • A sintaxe básica do comando DELETE segue a seguinte forma:

    DELETE FROM nome_da_tabele WHERE condição

  • Da mesma forma que em UPDATE a clausula WHERE indica em quais condições o comando deve ser executado.

    • Exemplo:

      DELETE FROM aluno WHERE id = 2

    • Verificando:

      SELECT * FROM aluno
      id nome sobrenome datanascimento cursoid
      1 Alice Gonçalves 1998-10-18 2
      3 Vitória Prestes 1997-02-15 3
      4 Alex Telles 1992-12-15 1
      • Repare que o aluno de id 2 foi deletado de banco

  1. Casos de exclusão:

    1. Exculsão simples:

      • Caso da exclusão anterior: uma linha deve ser removida com uma condição especifica, no exemplo a condição foi id = 2

    2. Exclusão utilizando 2 ou mais condições

      • Condições concatenadas pelos operadores AND e OR
      DELETE FROM nome_da_tabela WHERE nome="Ronaldo" AND sobrenome="Nazário"
      DELETE FROM nome_da_tabela WHERE username="sealmp4" OR username="Kopur"

    3. Exclusão com chave estrangeira

      • Pensando na tabela senacEstudo, imagine que o curso de Tecnico em Desenvolvimento de Sistemas deixou de existir, sendo assim, não há mais alunos desse curso e gostariamos de excluir do banco tanto o curso como os alunos matriculados no curso. Nesse caso há uma deleção com chave estrangeira, ou melhor, um DELETE Cascade. Isso significa que devemos excluir uma entrada que contem uma chave primária e também todas as entradas em outras tabelas que possuem essa chave primária como chave estrangeira
      • Esse comportamento não é permitido pelo SQL a não ser que, na criação de uma chave estrangeira seja especificado que o campo deve se comportar como DELETE CASCADE
      • Exemplo:
        • Se simplesmente executarmos
      DELETE FROM curso WHERE id = 1;

      Receberemos o seguinte erro:

      ERROR 1451 (23000) at line 2: Cannot delete or update a parent row: a foreign key constraint fails (`senacEstudo`.`aluno`, CONSTRAINT `aluno_ibfk_1` FOREIGN KEY (`curso_id`) REFERENCES `curso` (`id`))

      1. Criando uma tabela que permite o DELETE CASCADE

        • Só é possivel utilizar o DELETE CASCADE se criarmos uma tabela que permite o uso da função, ou se alterarmos uma tabela já existente para adicionar o comportamento
          • Criando a tabela
        CREATE TABLE aluno2 (
id int AUTO_INCREMENT,
nome varchar(30),
sobrenome varchar(30),
data_nascimento date,
curso_id int,
PRIMARY KEY (id),
FOREIGN KEY (curso_id) REFERENCES curso(id) ON DELETE CASCADE
);
        utilizaremos essa tabela para testar o DELETE CASCADE 
        INSERT INTO curso (id, nome) VALUES (12, 'Técnico em Informática');
    INSERT INTO aluno2 (nome, curso_id) VALUES ('Daltron', 12), ('Wilson', 12);

      2. Sobre o DELETE CASCADE

        Apesar de muito eficiente, o uso da exclusão em cascata pode comprometer a exclusão de dados muito sensíveis, já que todos os registros que estiverem referenciando com chave estrangeira o registro a ser excluído também serão apagados. Portanto, ainda falando da exclusão de registros referenciados por chaves estrangeiras, existem outras abordagens para esse cenário.

        • Formas indicadas
          • Atualizar todos os campos de cursoid, os steando para NULL antes de fazer uma exclusão:
        UPDATE aluno SET curso_id = NULL WHERE curso_id = 1;
    DELETE curso WHERE id = 1;

    4. Exclusão preservando o registro no banco de dados

      • Antes de continuar é importante adicionar alguns campos nas tabelas aluno e curso
      ALTER TABLE aluno ADD COLUMN status char(1) DEFAULT 'A';
ALTER TABLE curso ADD COLUMN status char(1) DEFAULT 'A';

      • Para preservar os registros em banco de dados deve-se evitar ao máximo a exclusão de dados. Para isso utilizaremos a coluna "status", para indicar a situação de um registro. No exemplo utilizaremos os Status
        • A: para ativo
        • I: para inativo
        • F: para um aluno que se formou
      • Agora, ao invés de deletar o registro do curso e do aluno podemos alterar os seus respectivos Status, preservando assim a informação no banco.
      • Suponha novamente que o curso de Tecnico em desenvolvimento de Sistemas não existe mais e precisamos alterar o status dos alunos que faziam esse curso

UPDATE aluno SET status = 'I' WHERE curso_id = 1

      • Isso também é util em situações onde apenas a tabela filha precisa ser alterada. Suponha que o aluno de id = 1 (Alice) se formou e o aluno de id = 2 (Vitória) desistiu do curso. Ainda assim é imporante manter a informação de que esses alunos um dia fizeram determinado curso, sendo assim podemos apenas alterar o campo "status" de cada aluno sem precisar alterar a tabela pai ou excluir a informação de que curso foi cursado por esse aluno

        UPDATE aluno SET status = 'F' WHERE id = 1;
UPDATE aluno SET status = 'I' WHERE id = 3;
