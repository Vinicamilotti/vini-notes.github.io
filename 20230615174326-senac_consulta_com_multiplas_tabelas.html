Consultando muitiplas tabelas utilizando WHERE

  • Pode-se unir as linhas da tabela de produtos com as linhas da tabela de marca utilizando para isso a cláusula WHERE. Confira o exemplo de consulta em SQL a seguir.

    SELECT Produtos.nomeProduto, Produtos.valorVendaProduto, Marca.nomeMarca FROM Produtos, Marca WHERE Produtos.idMarca = Marca.idMarca

  • Nesse caso o banco faz uma consulta as tabelas Produtos e Marcas, verifica em que casos o campo idMarca da tabela produtos é igual ao campo idMarca da tabela Marca e seleciona os campos associando as chaves

  • A forma genérica dessa forma de consulta é:

    SELECT tabela1.campoX, tabela2.campoY FROM tabela1, tabela2 WHERE tabela1.foreignKey = tabela2.primaryKey

  • Podem existir situações em que o mesmo nome de coluna esteja em ambas as tabelas. Para esse caso é preciso que as tabelas sejam diferenciadas no SELECT

    SELECT from Cliente.nome AS  NomeCliente, Cidade.nome AS Cidade FROM Cliente, Cidade WHERE Cliente.id = Cidade.id

    • Exemplo prático de utilização do ALIAS (AS)

      SELECT P.nomeProduto, P.valorVendaProduto AS Valor, M.nomeMarca FROM Produtos AS P, Marca as M WHERE P.idMarca = M.idMarca;

    A definição de alias pode ser feita com o uso da palavra-chave AS (como em Produto AS P), mas pode também ser definida diretamente sem esse termo (exemplo Produto P). É preferível, por clareza, utilizar sempre o termo AS ao definir alias.

Consultando dados em multiplas tabelas usando junção - JOIN

  • O método usando a calusula WHERE é um método mais antigo de consulta a diversas tabelas. A forma mais comum de consulta conjuntiva a multiplas tabelas é JOIN também conhecido como INNER JOIN

  • Sintaxex básica do operador JOIN

    SELECT * FROM tabela1 JOIN tabela2 ON tabela1.primary_key = tabela2.foreign_key
    • Destrinchando:
      • Após o FROM o comando descreve o nome da primeira tabela
      • O comando JOIN informa qual a outra tabela que deve ser utilizada
      • O comando ON informa a consulta em quais colunas os dados devem ser concatenados. O que geralmente envolve a chave estrangeira contida em uma tabela e a chave primária de outra (referenciada pela primeira)

  • Exemplo:

    SELECT P.nomeProduto, P.valorVendaProduto, M.nomeMarca FROM Produtos AS P JOIN Marca AS M ON P.idMarca=M.idMarca

  2. Filtrando dados de multiplas tabelas utilizando o JOIN em conjunto com o WHERE e ordenando através do comando ORDER BY

          • Pode ser utilizado em tipos DATE/TIME/DATETIME ordenando do mais velho ao mais recente

  3. Tipos de JOINS no SQL

    • Assim como visto em Senac: Modelagem conceitual do banco de dados em bancos de dados relacionais trabalhamos com teoria de conjuntos. Dessa forma, além do JOIN normal, que faz a intersecção entre duas tabelas ( A⋂B = {x:x ∈ A e x ∈ B}) também há a possibilidade de criarmos LEFT JOINS e RIGHT JOINS que retornam registros de uma tabela em determinada posição da relação

    1. LEFT JOIN

      • Retorna TODOS os registros da tabela a esquerda e APENAS os correspondetes da tabela a direita.

      • Sintaxe geral:

        SELECT coluna FROM tabelaESQUERDA JOIN tabelaDIREITA on tabelaESQUERDA.id1 = tabelaDIREITA.id2

      • Exemplificando:

        • Imagine que é necessário obter o total agrupado de vendas dos produtos, incluindo também os produtos que não tiveram nenhuma venda. Lembrando que:
          • A tabela produtos contem 7 produtos cadastrados
          • Apenas dois produtos tem registros na tabela de vendas - ver Senac: Consulta de dados
